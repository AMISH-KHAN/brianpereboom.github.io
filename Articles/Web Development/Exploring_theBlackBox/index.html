Why Event Loop

JavaScript is a single-threaded language, meaning that it can executes only one task at a time This might potentially lead to blocking execution if one task takes a significant amount of time to complete. fortunately JavaScript event loop enables it to execute such task without blocking the main thread making it a non-blocking language. To further undertstand what event loop is we need to under stand its architecture:
1. Call stack
2. Web API
3. Event Queue

Call Stack

Call Stack maintains the track of currently executing functions. It is a stack so it performs LIFO (Last In First Out) operations this means that the last function called is at the top of the Call stack.
when the function is called it get pushed into the call stack. After the fuction is completed it get popped off the stack so that next function can execute.
Call stack is Synchronous in nature so the function at the top needs to complete before calling the next function.

Web API

Web APIs are the functions and methods that are not the part of the JavaScript language but are provided by the web browser or JavaScript runtime enviroment(ex nodeJS) The execution of these function are done outside the JavaScript. setTimeOut() and fetch() are some of the exaples of Web API.

Event Queue

Whenever an Asynchronous task like setTimeOut() is done it returns a callback this callback doesnot directly pushed into the Call stack but instead it goes inside Event Queue.
Event Queue maintains all the callbacks provided by the Asynchronous function in FIFO(First In First Out) principle.

Event loop

Now we know that after the execution of Asynchronous task its provided callback does not directly goes into call stack but moves into the event queue but in order to completly execute a function must go in call stack. So when will this callback moves from event queue to call stack?
This is where Event Loop comes into the picture. It continously moniter the call stack and Whenever the stack is empty the callback is pushed into it.
This whole process allows JavaScript to run Asynchronous task without blocking the main thread.

Conclusion

Asynchronous nature of JavaScript is achieved through event loop. Hence Event Loop is sometimes called heart of javascript and it is important to undertstand because it reveales the magic of Handling async task.
